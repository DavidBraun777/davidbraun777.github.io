# Niche Work Playbooks

This folder is your execution system for building niche projects that do two
things at the same time:

1. Create completed assets you can sell from
2. Prove you are an operator who builds future-ready systems, not just a
   portfolio owner

These playbooks are grounded in your existing portfolio signals:

- Accessibility and non-profit delivery (VIFG)
- Cloud automation and infrastructure (AWS, Ansible, Docker, CI/CD)
- Security engineering (SAST/DAST, OAuth2/JWT, CSRF defense)
- Microservices and API integration (Spring Boot, REST/OpenAPI)
- Applied AI direction (MS in AI, serverless document/audio workflows)
- Cyber lab operations (Proxmox, Kubernetes, isolated testing)

## How To Use This Folder

1. Pick one niche from the playbooks
2. Build one flagship project with measurable outcomes
3. Publish proof assets on both domains:
   - summary on `dbraun.io`
   - full commercial case on business site
4. Package the work into a paid offer
5. Repeat with the next niche

## Deliverables Every Niche Project Must Produce

Each project should ship with:

1. Working demo (public or gated)
2. Architecture diagram
3. Technical write-up
4. Business case study with measurable outcomes
5. Offer page and CTA on business site
6. Cross-domain link pair between portfolio and business site

## Definition of "Done" for a Niche

A niche is considered ready for sales when:

1. One flagship project is fully built and documented
2. One paid offer exists (with clear scope and pricing range)
3. One discovery call script and qualification form are live
4. One public case study can be shared in outreach
