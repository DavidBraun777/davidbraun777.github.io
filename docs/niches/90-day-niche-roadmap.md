# 90-Day Niche Build Roadmap

This roadmap sequences niche projects so each one builds reusable credibility
and commercial leverage for the next.

## Primary Goal

By day 90, have:

1. Two completed flagship niche projects
2. Two sellable service offers backed by proof
3. A repeatable content-to-conversion system across both domains

## Phase 1 (Days 1-30): First Flagship + Offer

Recommended first niche:

- `01-accessibility-nonprofit-systems.md`

Deliverables:

1. Flagship build complete
2. Portfolio summary case study published
3. Business full case study published
4. Service page and booking CTA live
5. Cross-domain UTM tracking verified

## Phase 2 (Days 31-60): Second Flagship + System Reuse

Recommended second niche:

- `02-secure-api-platform-modernization.md` or
- `03-cloud-devops-automation-factory.md`

Deliverables:

1. Second flagship build complete
2. Reusable components shared across both niches
3. Offer ladder documented (starter/core/premium)
4. One comparison content piece:
   - "Which engagement is right for which team?"

## Phase 3 (Days 61-90): Scale Proof + Thought Leadership

Recommended third build direction:

- `04-ai-knowledge-operations-assistant.md` as a future-facing differentiator

Deliverables:

1. AI-focused pilot artifact with measurable outcomes
2. One flagship thought piece positioning David as a systems futurist
3. Unified "Work With Us" routing across both domains
4. Monthly reporting baseline for leads, conversions, and case-study impact

## Weekly Execution Cadence

1. Build progress (core implementation)
2. Documentation progress (architecture + runbook)
3. Content progress (portfolio summary + business conversion page)
4. Metrics progress (before/after and attribution)

## Risk Controls

1. Do not run more than one flagship build at a time.
2. Every build must end with publishable proof.
3. If no measurable metric can be captured, narrow scope until metric exists.
4. Prioritize revenue-connected niches before exploratory niches.

## "Top-Notch" Standard Checklist

Before marking any niche complete:

1. Demo is stable
2. Architecture diagram exists
3. Case study has measurable outcomes
4. Service offer is published and priced
5. Cross-domain links are live and tracked
6. Discovery CTA is functional
