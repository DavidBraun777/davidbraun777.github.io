# Niche 01: Accessibility Nonprofit Systems

## Positioning

"I build accessibility-first digital systems for mission-driven organizations
that need reliability, trust, and operational simplicity."

This niche aligns with:

- VIFG project delivery
- React/Vite/Next.js implementation
- AWS + Docker + GitHub Actions + Nginx operations
- Accessibility-first mindset

## Ideal Buyer

- Non-profit leaders (Executive Director, Operations, Programs)
- Small mission-driven organizations with outdated web systems
- Teams that need donations/events/content workflows without enterprise
  complexity

## Flagship Project To Build

Project name:

- Mission Access Platform

Scope:

1. Accessible public site (WCAG-conscious baseline)
2. Donation flow
3. Event and volunteer workflow
4. Admin dashboard for updates
5. CI/CD deployment and backup strategy

## Productized Offer Ladder

1. Starter: Accessibility Audit + Quick Wins (1-2 weeks)
2. Core: Accessibility Site Rebuild + Operational Handoff (4-8 weeks)
3. Premium: Multi-site nonprofit platform + training + maintenance

## 30-Day Build Plan

Week 1:

- Create requirements brief and IA
- Define component accessibility standards
- Implement core page templates and CMS/data model

Week 2:

- Build donation and event modules
- Add form validation, alerts, and accessibility states
- Configure CI/CD and staging environment

Week 3:

- Add admin workflows, content editing flow, and QA checklist
- Conduct screen reader and keyboard-only pass
- Run Lighthouse and accessibility audits

Week 4:

- Finalize documentation and launch checklist
- Capture before/after metrics
- Publish case study assets

## Proof Assets To Publish

Business site:

- Full case study with measurable impact
- Offer page with scope/options

1. Accessibility score improvements
