# Niche Selection Framework

Use this framework before committing to the next build. It prevents random
project drift and keeps your work tied to revenue potential.

## Scoring Model (1-5 per criterion)

Score each niche from 1 (low) to 5 (high):

| Criterion | Weight | What to look for |
| --- | ---: | --- |
| Existing credibility | 25% | Already supported by your portfolio/work history |
| Buyer demand | 25% | Frequent pain point with budget attached |
| Differentiation | 20% | Your edge is hard to copy |
| Time to proof | 15% | Can ship flagship proof in 2-6 weeks |
| Reusability | 10% | Components can be reused in future client work |
| Offer clarity | 5% | Easy to package into a paid engagement |

Weighted score formula:

`(credibility * .25) + (demand * .25) + (differentiation * .20) + (time * .15) + (reusability * .10) + (offer * .05)`

Target:

- Prioritize niches scoring `>= 4.0`

## Fast Decision Rules

Choose the niche if all are true:

1. You can explain the client pain in one sentence
2. You can build a demo within 30 days
3. You can define a paid discovery engagement from it
4. You can produce one measurable case study metric
